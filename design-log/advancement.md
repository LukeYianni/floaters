# Advancement — Design Log

## Specialism XP Triggers

### What was decided

Specialism XP triggers were removed. XP comes from two sources only: **failure** and **end-of-session questions**. There is no in-play XP trigger tied to a character's Specialism.

### Why Specialism triggers were cut

Three reasons, considered together:

**1. The starting abilities already do this job.**
Most Specialism starting abilities already fire an immediate reward when the character does their thing effectively — typically 1 Edge to an ally. That Edge *is* the in-play signal that the method mattered. Adding XP on top of Edge on top of the ability effect created three things firing from one moment, diluting each signal.

**2. The failure XP has an elegant implicit logic that Specialism triggers break.**
Failure XP is self-balancing: you fail more where you're weak, advance faster there. Your Specialism is where you *succeed* — so you earn less failure XP in it naturally. This pushes characters toward their edges, not their comfort zones. A Specialism trigger would reward succeeding at your forte — a different economic signal that competes with the failure logic rather than complementing it.

**3. The Instinct already handles character-fantasy at end of session.**
The end-of-session Instinct question rewards the behavioral texture of who this character is. A Specialism trigger would reward capability expressed effectively — a shallower version of the same thing, crowding out the Instinct's signal without adding anything distinct.

### What to do if a third XP source is added later

If the system needs a third XP source, it should be structural and crew-level (e.g. a Crew Type trigger) rather than an individual Specialism fire. That would add a genuinely distinct texture — collective identity rewarded — without duplicating what failure XP and Instinct already do.
